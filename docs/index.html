At bot startup, a single __schema GraphQL introspection query downloads the complete API schema (203+ types). 🗺️ N-Level Navigation

Navigate through the compendium via a fully inline keyboard — never leave Telegram:

📖 Wiki → Category → Paginated List → Item Detail → 📂 Sub-entities → …

Each item detail page auto-discovers its navigable sub-entities (e.g., a Class shows its proficiencies, saving throws, spells, and subclasses; a Race shows traits, languages, subraces; a Spell shows the classes that can cast it). These relationships are computed from the schema — no manual configuration needed.

⚙️ Technical Highlights

  • Union type support — uses __typename + inline fragments to handle polymorphic types like AnyEquipment
  • Server-side pagination — top-level lists use skip/limit GraphQL arguments; sub-lists use client-side slicing
  • Partial error handling — if the API returns errors alongside data, the bot still shows available content and logs a warning
  • 15-second HTTP timeout via httpx.AsyncClient

⚔️ Character Management Gestione Personaggio

A complete digital D&D character sheet, fully managed via Telegram inline keyboards and a persistent conversation flow. Characters are stored in a local SQLite database and survive bot restarts.

🧙 Character Creation & Selection

  • Create multiple characters per Telegram account
  • Creation wizard: enter name → choose class (12 predefined D&D 5e classes or free text) → optionally set subclass → set starting level
  • Switch between characters at any time from the selection screen
  • Delete characters with confirmation prompt
  • Character state (active menu, mid-flow step) persists across bot restarts via PicklePersistence

❤️ Hit Points & Armor Class

  • ⬆️ Set max HP
  • 📉 Apply damage
  • 💚 Apply healing
  • 🎯 Set current HP directly
  • 🌙 Short rest (restores short-rest class resources)
  • ☀️ Long rest (full HP, resets long-rest resources, drops concentration)
  • 🛡️ Base AC, shield AC, magic AC bonus — combined automatically

Every HP and AC change instantly syncs the live party message (if a party session is active).

📊 Ability Scores & Skills

Set all six ability scores (STR, DEX, CON, INT, WIS, CHA). Modifiers are computed automatically using the standard D&D formula (score − 10) ÷ 2.

18 standard D&D 5e skills, each linked to its ability score:

  • Deception (CHA)

For each skill: toggle proficiency on/off, view the computed bonus (ability mod + proficiency bonus), and roll d20 + bonus directly from the detail screen. Results are saved in history. Proficiency bonus scales with total level: +2 (lv 1–4) → +3 (lv 5–8) → +4 (lv 9–12) → +5 (lv 13–16) → +6 (lv 17–20).

🔮 Spells & Spell Slots

A full spell management system. Supports two navigation modes (configurable in Settings): Browse by level (level picker → filtered list) or Flat list (all spells scrollable).

  • Learn spells — quick-add: name → level → concentration? (all other fields editable later)
  • Edit any field — level, casting time, range/area, components, duration, concentration, ritual, attack/save, description, higher-level description
  • Use spell — consumes a spell slot of the chosen level from your slots pool
  • Concentration tracking — only one concentration spell active at a time; auto-activated when casting; dropped on rest
  • Concentration saving throw — DC = max(10, damage ÷ 2); roll d20 + CON modifier; natural 1 always fails, natural 20 always succeeds
  • Fuzzy spell search — type a partial name and get ranked results instantly using rapidfuzz
  • Pin spells — pinned spells appear in the character main menu summary
  • Forget spells — remove a spell from your sheet
  • Manage spell slots — add, use, restore, and remove slot levels independently of spells

🛡️ Group Party Party di Gruppo

Bring your D&D session to Telegram! The party feature displays a live-updated status card for every active character in your group — refreshed automatically whenever anything changes.

🚀 Starting a Party Session

  1. Send /party in your Telegram group
  2. Choose display mode:
    • 🌐 Public — the party status message is posted in the group for everyone to see
    • 🔒 Private (GM mode) — the bot sends the party status directly to the Game Master's private chat
  3. The session lasts 48 hours with a live countdown. End it early with /party_stop

👥 Who Appears in the Party?

  • Each player marks one character as Party Active from their character settings
  • If a player has exactly one character and hasn't marked it active, it is included automatically
  • The bot tracks all users who write in the group to know who belongs to the session

🔄 Real-Time Updates

The party message is automatically refreshed whenever a party member changes:

  • ❤️ HP (damage / heal / rest)
  • 🛡️ AC (base / shield / equip armor)
  • 🌀 Conditions (toggle / exhaustion)
  • 🎲 Any dice roll

Updates are fire-and-forget (never block the user's response) and handle stale/deleted messages gracefully.

📋 Party Message Format

Each character row in the party message shows:

  • Name & class/level (e.g. Thorin — Warrior 5 / Paladin 3)
  • HP bar — visual progress bar (████░░░░ 42/60) colour-coded by health percentage
  • AC — current armour class
  • Active conditions — compact emoji list (hidden when none)
  • Last dice roll — most recent roll label and result (hidden when no history)

The message header shows the group name and the remaining session time.

Porta la tua sessione D&D su Telegram! La funzione party mostra una scheda di stato live per ogni personaggio attivo nel gruppo — aggiornata automaticamente ogni volta che qualcosa cambia.

🚀 Avviare una Sessione Party

  1. Invia /party nel tuo gruppo Telegram
  2. Scegli la modalità di visualizzazione:
    • 🌐 Pubblica — il messaggio di stato party viene pubblicato nel gruppo, visibile a tutti
    • 🔒 Privata (modalità Master) — il bot invia lo stato party direttamente nella chat privata del Game Master
  3. La sessione dura 48 ore con un conto alla rovescia live. Terminala prima con /party_stop

👥 Chi Appare nel Party?

  • Ogni giocatore marca un personaggio come Party Attivo dalle impostazioni del personaggio
  • Se un giocatore ha esattamente un personaggio e non l'ha marcato attivo, viene incluso automaticamente
  • Il bot tiene traccia di tutti gli utenti che scrivono nel gruppo per sapere chi appartiene alla sessione

🔄 Aggiornamenti in Tempo Reale

Il messaggio party viene aggiornato automaticamente ogni volta che un membro del party cambia:

  • ❤️ HP (danno / cura / riposo)
  • 🛡️ CA (base / scudo / equipaggia armatura)
  • 🌀 Condizioni (toggle / esaurimento)
  • 🎲 Qualsiasi tiro di dado

Gli aggiornamenti sono fire-and-forget (non bloccano mai la risposta dell'utente) e gestiscono in modo elegante i messaggi obsoleti/eliminati.

📋 Formato del Messaggio Party

Ogni riga personaggio nel messaggio party mostra:

  • Nome e classe/livello (es. Thorin — Guerriero 5 / Paladino 3)
  • Barra HP — barra di progresso visiva (████░░░░ 42/60) con colore in base alla percentuale di salute
  • CA — classe armatura corrente
  • Condizioni attive — lista emoji compatta (nascosta se nessuna)
  • Ultimo tiro di dado — etichetta e risultato dell'ultimo tiro (nascosto se nessuno storico)

L'intestazione del messaggio mostra il nome del gruppo e il tempo rimanente della sessione.

cd DnD-Adventurers-Tome-TGBot
python -m venv .venv
# Windows:
.\.venv\Scripts\Activate.ps1
# Linux / macOS:
source .venv/bin/activate
pip install -r requirements.txt

Configuration

Create a .env file in the project root:

BOT_TOKEN=your_bot_token_here
DEV_CHAT_ID=optional_telegram_chat_id
DB_PATH=data/dnd_bot.db
VariableRequiredDescription
BOT_TOKEN✅ YesTelegram bot token from @BotFather
DEV_CHAT_IDNoYour Telegram chat ID — unhandled exceptions are forwarded here as full tracebacks
DB_PATHNoSQLite DB file path (default: data/dnd_bot.db)

Run

python -m bot.main
💡 Tip: Set DEV_CHAT_ID to receive full Python tracebacks for unhandled errors directly in your Telegram DM — extremely useful during development.

🌍 Internationalisation (i18n) Internazionalizzazione (i18n)

  • User language is detected from update.effective_user.language_code (e.g., it-IT → it)
  • Supported locales out of the box: Italian (default) and English
  • All user-facing strings live in bot/locales/it.yaml and bot/locales/en.yaml — never hardcoded
  • Fallback chain: user_lang → "it" → key itself
  • Interpolation with named kwargs: translator.t("character.hp.title", lang=lang, current=45, max=60)
  • Adding a new language is as simple as creating a new <code>.yaml file in bot/locales/
  • Hot-reload: the bot checks YAML file modification times every 5 minutes and reloads changed files — no restart needed
  • La lingua dell'utente viene rilevata da update.effective_user.language_code (es. it-IT → it)
  • Locali supportate: Italiano (default) e Inglese
  • Tutte le stringhe visibili all'utente si trovano in bot/locales/it.yaml e bot/locales/en.yaml — mai hardcoded
  • Catena di fallback: lingua_utente → "it" → chiave stessa
  • Interpolazione con kwargs nominativi: translator.t("character.hp.title", lang=lang, current=45, max=60)
  • Aggiungere una nuova lingua è semplice come creare un nuovo file <codice>.yaml in bot/locales/
  • Hot-reload: il bot controlla i timestamp di modifica dei file YAML ogni 5 minuti e ricarica i file cambiati — senza riavvio
